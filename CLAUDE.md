# 股票数据维护项目

## 项目概述

自动维护全市场股票日线、15分钟、30分钟、60分钟、月线数据（不复权 + 换手率），并维护全市场股票基本信息快照（总市值、流通市值等）、上证指数日线、15分钟、30分钟、60分钟、月线缓存以及全市场除权/股本变更缓存；同时维护申万行业板块数据与申万行业成分股缓存。个股行情数据源为 BaoStock，基本信息快照当前使用腾讯批量行情接口，上证指数分钟线使用东财，除权数据源使用当前项目内置的 AkShare/东方财富抓取链路。

- **数据规模**: 5,170只股票，15,851,494行历史数据
- **存储格式**: Parquet（442.4 MB）
- **数据范围**: 1990-12-19 至今
- **更新方式**: 每天18:00自动更新，错过后自动补跑未完成交易日

- `数据维护/`：日常数据更新与调度相关脚本的组织视图
- `策略研究/`：策略、回测、版本存档与测试的组织视图
- `工具排查/`：下载进度、服务调试、探索脚本和历史恢复脚本的组织视图
- 这几个目录当前以软链接方式组织，源码仍以根目录文件为主，方便按需求分区又不影响现有导入与测试

## 重要配置

### 数据格式
- **复权方式**: `复权方式 = "3"` (不复权，原始价格)
  - 1 = 后复权
  - 2 = 前复权
  - 3 = 不复权（当前设置）
- **是否同步CSV**: `同步更新CSV = False` (仅更新Parquet，速度快10倍)

### 数据列定义
```
stock_code, stock_name, date, open, high, low, close,
volume, amount, outstanding_share, turnover
```

## 使用说明

### 日常更新
```bash
python 智能更新.py
```
- 自动检测缺失交易日
- 从BaoStock并发补齐（多进程）
- 同时更新Parquet文件

### 股票基本信息快照更新
```bash
python3 更新基本信息.py
```

- 默认输出到 `数据/股票基本信息.parquet`
- 当前快照源使用腾讯批量行情接口（`qt.gtimg.cn`）
- 当前输出字段：
  - `stock_code, stock_name, date, close, change_pct, change_amount, volume, amount`
  - `amplitude, high, low, open, prev_close, volume_ratio, turnover`
  - `pe_dynamic, pb, total_market_cap, float_market_cap`
  - `speed, change_5min_pct, change_60d_pct, change_ytd_pct`
  - `total_shares, float_shares`
- `total_shares` / `float_shares` 为按 `总市值 ÷ 最新价`、`流通市值 ÷ 最新价` 反推的快照值
- 自动调度会把它作为独立管线纳入每日更新

### 分钟线派生
```bash
python3 派生分钟数据.py            # 按现有15分钟文件补建/补齐30和60分钟
python3 派生分钟数据.py --freqs 30 # 只补30分钟
python3 派生分钟数据.py --force-full
```

- 30分钟、60分钟均由 `数据/全市场15分钟数据.parquet` 聚合得到
- 输出字段与15分钟完全一致：`stock_code, stock_name, date, time, open, high, low, close, volume, amount`
- 自动调度会在15分钟更新后继续校验并补齐30/60分钟文件

### 上证指数多周期更新
```bash
python3 智能更新上证指数.py
python3 智能更新上证指数.py --dry-run
```

- 默认输出到：
  - `数据/上证指数日线数据.parquet`
  - `数据/上证指数15分钟数据.parquet`
  - `数据/上证指数30分钟数据.parquet`
  - `数据/上证指数60分钟数据.parquet`
  - `数据/上证指数月线数据.parquet`
- 日线使用 BaoStock 的 `sh.000001`，15/30/60 分钟使用东财分钟接口
- 分钟文件会合并本地已有数据与东财可用窗口，首次切换源后会把更早历史补进来
- 月线由日线聚合生成，确保当前月会随日线一起刷新
- 自动调度会把它作为独立的数据组一起补跑和校验

### 除权数据更新
```bash
python3 更新除权数据.py
python3 更新除权数据.py --codes 000001,600000
python3 更新除权数据.py --limit 50
```

- 默认输出到 `数据/除权数据/`
- 当前维护 3 份文件：
  - `corporate_actions.parquet`: `stock_code, ex_date, bonus_per10, transfer_per10, cash_div_per10, rights_per10, rights_price`
  - `share_changes.parquet`: `stock_code, change_date, float_shares`
  - `fetch_summary.json`: 记录生成时间、股票数、事件数和回退统计
- 抓取逻辑已内置在当前项目：`更新除权数据.py` + `除权数据抓取.py`
- 自动调度会在日线更新完成后执行一次除权缓存更新，确保当前项目下也有最新除权数据可直接复用

### 行业板块数据更新
```bash
python3 更新行业数据.py
python3 更新行业数据.py --only sw
python3 更新行业数据.py --only cons
```

- `更新行业数据.py` 维护申万一级/二级行业日线，以及申万行业成分股映射
- `--only sw` 会同时更新申万一级和二级行业日线
- 这些数据文件统一写入 `数据/`，便于板块研究和横向对比

### 自动任务管理
```bash
python3 自动任务.py --安装    # 安装每日18:00自动更新
python3 自动任务.py --状态    # 查看任务状态
python3 自动任务.py --卸载    # 卸载自动任务
```

- macOS: 使用 launchd 通过系统 shell 拉起 Python 调度
- macOS: launchd 的 `WorkingDirectory` 固定为 `~/`，stdout/stderr 写入 `~/Library/Logs/com.hongjian.stockdata.autoupdate.log`，避免后台任务因 `Documents` 目录权限导致 `Operation not permitted`
- Windows: 使用 schtasks 调度
- 执行顺序：日线更新 -> 基本信息快照更新 -> 上证指数多周期更新 -> 除权缓存更新 -> 月线更新 -> 15分钟增量更新 -> 30/60分钟派生
- 容错机制：全天整点/半点自然调度（脚本内部按交易日自动补跑，若错过18:00会自动补跑未完成交易日/月）
- 若15分钟写回后逐股校验仍未补齐到目标交易日，脚本会返回非零，调度器不会记为成功
- 若15分钟在同一交易日已失败且输入快照未变化，调度器会先跳过重复重试，等日线或15分钟文件变化后再重试，避免半小时一轮死循环
- 若本轮只是自然巡检且没有实际更新/修复，调度器只记日志，不再重复发送“自动更新完成”通知
- 若15分钟因当日失败被跳过，后续自然调度不会继续重跑30/60分钟派生，避免把失败轮次误报成“已完成”
- `自动调度更新.py` 当前只负责核心行情链路；行业/板块数据需手动运行 `更新行业数据.py`
- 15分钟补齐目标按日线最近有成交日期判断；日线末尾若是停牌/零成交占位日，不要求分钟线追到该日期
- 若任一子任务失败，`自动调度更新.py` 会整体返回非零，便于在 `launchctl` 状态里直接识别失败
- 子脚本统一带超时保护；若子脚本挂死，调度器会按失败处理并释放调度锁
- 调度状态文件改为原子写入，避免异常中断时写出半截 JSON
- 通知失败不再静默吞掉，调度日志会记录通知异常或非零返回
- macOS 安装后不会立即触发，等待下一次自然调度

## ⚠️ 重要注意事项

1. **BaoStock互斥**
   - 智能更新脚本内置互斥锁
   - 不要同时运行多个BaoStock脚本（会互相影响速度）
   - 锁文件在系统临时目录：`<temp>/baostock.lock`

2. **数据完整性**
   - 原始CSV文件在 `D:\Quant\Data\stock_data_sina_raw\` (只读，不会修改)
   - 主数据文件：`数据/全市场历史数据.parquet`
   - 分钟派生文件：`数据/全市场30分钟数据.parquet`、`数据/全市场60分钟数据.parquet`
   - 上证指数文件：`数据/上证指数日线数据.parquet`、`数据/上证指数15分钟数据.parquet`、`数据/上证指数30分钟数据.parquet`、`数据/上证指数60分钟数据.parquet`、`数据/上证指数月线数据.parquet`
   - 基本信息文件：`数据/股票基本信息.parquet`
   - 除权缓存文件：`数据/除权数据/corporate_actions.parquet`、`数据/除权数据/share_changes.parquet`
   - 行业/板块文件：`数据/申万行业日线数据.parquet`、`数据/申万二级行业日线数据.parquet`、`数据/申万行业成分股.parquet`
   - 默认不同步更新CSV以提升速度

3. **复权数据切换**
   - 如需复权数据，修改 `配置.py` 中 `复权方式 = "2"` (前复权)

## 故障排查

### 更新失败
1. 查看 `更新日志.txt`
2. 检查网络连接
3. 确认BaoStock服务是否正常

### 计划任务未运行
1. Windows: 打开「任务计划程序」查看任务状态
2. macOS: 执行 `python3 自动任务.py --状态`
3. 检查 `进度/自动调度日志.txt` 和 `进度/自动任务_launchd.log`
4. 如 macOS 的 `launchd` 任务异常，重新执行 `python3 自动任务.py --安装`
5. Windows 手动触发测试：`Start-ScheduledTask -TaskName "股票数据每日更新"`
