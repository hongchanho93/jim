# 15分钟K线数据下载使用说明

## 概述

本系统用于高性能下载全市场股票的15分钟K线数据，采用10线程并发下载，预计耗时1.5-2小时。

### 数据特点

- **价格类型**: 不复权（与日线数据一致）
- **时间范围**: 2024-01-01 至今（约13个月）
- **数据字段**: 9个基础字段（无换手率，因BaoStock分钟线接口不支持）
- **股票数量**: 5,000+ 只
- **预计数据量**: 2,000-2,500万行
- **文件大小**: 500-700 MB

### 输出字段

| 字段 | 说明 |
|------|------|
| stock_code | 股票代码 |
| stock_name | 股票名称 |
| date | 日期 YYYY-MM-DD |
| time | 时间 HH:MM:SS |
| open | 开盘价（不复权） |
| high | 最高价（不复权） |
| low | 最低价（不复权） |
| close | 收盘价（不复权） |
| volume | 成交量 |
| amount | 成交额（元） |

## 常见问题

### Q1: 下载速度很慢怎么办？

**可能原因**：
- BaoStock API 限流
- 网络不稳定

**解决方案**：
1. 降低并发数（改为 5 线程）
2. 非交易时段运行（晚上/周末）
3. 耐心等待，脚本有自动重试机制

### Q2: 部分股票下载失败怎么办？

**正常情况**：
- 失败率 < 5% 是正常的
- 新股、退市股可能无数据

**处理方法**：
- 脚本会记录失败股票
- 可以手动重新运行脚本（断点续传会跳过成功的）

### Q3: 如何清理临时文件？

下载完成后，可以删除临时文件节省空间：

```bash
rm -rf "/Users/hongjian/Documents/CODEX项目/股票数据维护/数据/临时15分钟"
```

**注意**：确保最终文件 `全市场15分钟数据.parquet` 已生成再删除。

### Q4: 为什么没有换手率字段？

BaoStock 的分钟线接口（`query_history_k_data_plus`）不支持换手率（`turn`）字段，只有日线接口支持。

**替代方案**：
- 从日线数据估算：`15分钟换手率 ≈ 日线换手率 / 16`
- 使用其他付费数据源（如 Tushare Pro）

### Q5: 如何增量更新数据？

当前脚本下载全量历史数据（2024年至今）。如需每日增量更新，可以：

1. 修改 `开始日期` 为昨天
2. 运行脚本下载最新数据
3. 手动合并到现有数据文件

**未来优化**：可参考 `智能更新.py` 开发增量更新脚本。

## 技术细节

### 并发策略

- **线程池**: `ThreadPoolExecutor`（10线程）
- **任务分配**: 动态分配（`as_completed`）
- **重试机制**: 最多3次，指数退避
- **限流保护**: 自动检测并延迟重试

### 数据存储

- **格式**: Parquet (Apache Arrow)
- **压缩**: Snappy
- **分批保存**: 每100只股票保存一次
- **去重策略**: 按 `(stock_code, date, time)` 去重

### 断点续传

- **进度记录**: JSON格式
- **记录内容**: 已完成股票代码列表
- **恢复机制**: 自动跳过已完成股票

## 数据清洗与三周期更新

### 一次性全量清洗

首次建议先执行：

```bash
python3 数据清洗.py --dry-run
python3 数据清洗.py
```

说明：
- `turnover` 会统一为百分比数值（`0.35` 表示 `0.35%`）；
- 保留停牌/零成交记录，仅做字段标准化；
- 15分钟文件使用分块清洗 + 低内存合并，避免大内存峰值。

### 日常增量更新（推荐）

按顺序执行（或交给自动调度）：

```bash
python3 智能更新.py
python3 智能更新月线.py
python3 智能更新15分钟.py
```

三个更新脚本统一支持：

```bash
python3 智能更新.py --dry-run --limit-stocks 20
python3 智能更新月线.py --dry-run --limit-stocks 20
python3 智能更新15分钟.py --dry-run --limit-stocks 20
```

### 自动调度行为

`自动调度更新.py` 现在顺序执行：
1. 日线更新
2. 月线更新
3. 15分钟更新

调度幂等规则：
- 不仅检查“目标交易日/月是否完成”；
- 还会检查三份数据文件的快照（`size` + `mtime`）；
- 若你手动替换了数据文件，即使同一交易日也会自动触发补齐。
- 15分钟更新还会记录它所依赖的日线快照；只要日线在同一交易日后续又补进了新数据，15分钟会重新校验并补跑，避免“显示更新完成但分钟线实际仍缺失”。
- `launchd` 采用全天整点/半点自然调度；18:00后按当日最新完整交易日更新，若错过18:00，后续调度会自动补跑未完成交易日/月。
- 若15分钟文件写回后仍有股票落后于目标交易日，脚本会返回非零，避免调度器误记为成功。
- 若同一交易日的15分钟补跑已失败且文件快照没有变化，后续自然调度会先跳过，不再每半小时重复跑同一批已知失败数据。
- 15分钟“是否最新”的判定以日线最近有成交的日期为准；若日线末尾是停牌/零成交占位日，不会再误判为分钟线缺失。

### 常见恢复操作

1. **只体检不落盘**  
`python3 数据清洗.py --dry-run`

2. **只清洗某一类数据**  
`python3 数据清洗.py --only daily --dry-run`  
`python3 数据清洗.py --only monthly --dry-run`  
`python3 数据清洗.py --only 15m --dry-run`

3. **15分钟历史缺口补齐（先小范围试跑）**  
`python3 智能更新15分钟.py --dry-run --limit-stocks 50`  
确认后再去掉 `--dry-run` / `--limit-stocks` 全量执行。
